  REGISTRATION WORKFLOW (Updated)
================================================================================

NURSE FLOW:
  1. Register
     ↓
  2. Receive approval email (admin sends)
     ↓
  3. Click "Claim Ticket"
     ↓
  4. Receive QR code email
     ↓
  5. Check in at event with QR code

ADMIN FLOW:
  1. View pending approvals
  2. Review nurse information
  3. Click "Approve"
     → Sets is_verified=true
     → Sends approval email
     → Sets email_sent=true
  4. Monitor workflow stats
  5. Resend emails if needed

================================================================================
  API CHANGES SUMMARY
================================================================================

MODIFIED ENDPOINTS:

  POST /api/nrpx/register
    • Improved rate limiting (IP 3/hr + Domain 1/day)
    • Better validation order (validate before rate limit)
    • Status: NOT breaking

  GET /api/nrpx/my-registration
    • REMOVED: ticket_code (never exposed via API)
    • ADDED: status field (pending_approval, approved_ready_to_claim, etc)
    • ADDED: email_sent, is_verified fields (better UX)
    • Status: BREAKING CHANGE (frontend needs update)

  POST /api/nrpx/claim-ticket
    • ADDED: email_sent validation check
    • Better error messages
    • Status: NOT breaking

  POST /api/admin/nrpx/approve/:id
    • Improved error handling
    • Always mark email_sent (even if email fails)
    • Better feedback on email failures
    • Status: NOT breaking

  GET /api/nrpx/stats
    • ADDED: ticketsClaimed, stillToCheckin fields
    • Status: NOT breaking (backward compatible)

NEW ENDPOINTS:

  GET /api/admin/nrpx/workflow-stats ⭐ NEW
    • Shows: pending approvals, ready to claim, tickets claimed, checked in
    • Shows: percentages for dashboard visualization
    • Use: admin dashboard to monitor pipeline
